# Задача 1. Теоретические основы классических алгоритмов обучения

## 1.1 Введение в постановку задачи

Задача обучения с учителем формулируется как поиск отображения \(f: \mathcal{X} \rightarrow \mathcal{Y}\) по обучающей выборке \(\{(x_i, y_i)\}_{i=1}^{n}\), минимизирующего ожидаемый риск. В классификации \(\mathcal{Y}\) — конечное множество меток. Качество оценивается функцией потерь и обобщающей способностью на тестовой выборке, не участвовавшей в обучении. Классические алгоритмы различаются индуктивными предпосылками: линейные модели предполагают разделимость гиперплоскостью, деревья — кусочно-постоянные аппроксимации, метод опорных векторов — максимизацию зазора, k-NN — локальную гладкость в пространстве признаков. Понимание этих предпосылок критично для выбора модели и диагностики переобучения и недообучения.

## 1.2 k-ближайших соседей (k-NN)

Метод k-NN относится к ленивому (instance-based) обучению: фаза «обучения» сводится к запоминанию обучающих объектов. Для запроса \(x\) вычисляются расстояния \(d(x, x_i)\) до всех обучающих точек (часто евклидово или по Минковскому), выбираются \(k\) объектов с наименьшим расстоянием, и предсказание — мажоритарный класс среди их меток. При весах, обратно пропорциональных расстоянию, вклад ближних соседей усиливается. Математически: \(\hat{y} = \mathrm{mode}\{y_i : x_i \in N_k(x)\}\). Сложность обучения \(O(1)\), инференса \(O(nd)\) на выборке из \(n\) объектов и \(d\) признаков — узкое место на больших данных. Преимущества: простота, отсутствие явного обучения, естественная мультиклассовость. Ограничения: чувствительность к масштабу признаков (необходима нормализация), проклятие размерности, шумные метки соседей. Гиперпараметр \(k\) управляет смещением-дисперсией: малый \(k\) — высокая дисперсия, большой — сглаживание границ. Лучшие сценарии: небольшие и средние табличные данные после стандартизации, базовые линии.

## 1.3 Решающие деревья и случайный лес

Дерево решений рекурсивно делит пространство признаков по правилам \(x_j \leq t\), минимизируя критерий неоднородности (индекс Джини \(G = 1 - \sum_k p_k^2\) или энтропия). Лист предсказывает мажоритарный класс. Интерпретируемость высока, но одно дерево склонно к переобучению. Случайный лес (Random Forest) строит ансамбль из \(T\) деревьев на бутстрап-подвыборках с случайным подмножеством признаков на каждом разбиении; итоговое решение — голосование. Это снижает дисперсию при сохранении низкого смещения отдельных деревьев. Обучение масштабируется как \(O(T \cdot n \log n \cdot d)\) при жадных сплитах. Гиперпараметры: `max_depth`, `min_samples_split`, `n_estimators`, `max_features`. Лес устойчив к выбросам и смешанным типам признаков после кодирования; важность признаков доступна через среднее уменьшение неоднородности. Ограничения: больший размер модели, слабее интерпретируемость, чем у одного дерева.

## 1.4 Метод опорных векторов (SVM)

SVM ищет разделяющую гиперплоскость \(w^\top x + b = 0\) с максимальным зазором между классами. Для линейно неразделимых данных используется ядерный трюк: \(K(x_i, x_j)\) неявно отображает точки в пространство большей размерности. Оптимизация — квадратичная задача с ограничениями; решение зависит только от опорных векторов. Решающая функция: \(f(x) = \mathrm{sign}(\sum_i \alpha_i y_i K(x_i, x) + b)\). Параметр \(C\) балансирует ширину зазора и штраф за ошибки; \(\gamma\) в RBF-ядре задаёт «радиус» влияния. Преимущества: эффективность в средних размерностях, сильная теория обобщения. Недостатки: квадратичная/кубическая сложность по \(n\) для точных солверов, необходимость масштабирования признаков, калибровка вероятностей отдельно (Platt scaling). Хорошо подходит для табличных данных среднего объёма и текстов с линейным ядром.

## 1.5 Логистическая регрессия

Логистическая регрессия моделирует апостериорную вероятность класса через сигмоиду (бинарный случай) или softmax (мультикласс): \(P(y=k|x) = \mathrm{softmax}(Wx)_k\). Обучение — максимизация лог-правдоподобия (кросс-энтропия) с возможной L2-регуляризацией \(\frac{\lambda}{2}\|W\|^2\). Функция выпуклая; оптимизация градиентными методами (L-BFGS, SAGA). Правило решения: \(\hat{y} = \arg\max_k P(y=k|x)\). Сложность обучения линейна по числу итераций и признаков; инференс — \(O(d)\). Преимущества: скорость, калиброванные вероятности, интерпретируемые веса. Недостатки: линейная граница в исходных признаках без feature engineering. Является сильным базовым методом и компонентом стекинга.

## 1.6 Наивный байесовский классификатор

Наивный Байес предполагает условную независимость признаков при фиксированном классе: \(P(y|x) \propto P(y)\prod_j P(x_j|y)\). Для непрерывных признаков — гауссовы плотности; для счётных — мультиномиальное распределение. Обучение сводится к оценке частот и параметров по классам — \(O(nd)\). Инференс быстрый. Предпосылка независимости редко выполняется, но классификатор устойчив на текстовых данных (мешок слов) и служит конкурентным baseline. Сглаживание Лапласа предотвращает нулевые вероятности.
