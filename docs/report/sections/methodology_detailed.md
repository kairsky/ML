# Детальная методология экспериментов

## Программная архитектура

Проект организован как installable-пакет `src` с разделением ответственности: `data` (загрузка и предобработка), `classical_ml` (обучение и оценка), `dimensionality` (PCA/UMAP/t-SNE/LDA), `deep_learning` (Keras MLP/CNN, PyTorch transfer), `visualization` (единый PlotManager), `utils` (конфигурация, логи, seed). Скрипты в `scripts/` — тонкие CLI-обёртки, вызываемые из notebooks. Такой дизайн соответствует принципу DRY и облегчает unit-тестирование отдельных компонентов.

## Протокол разбиения данных

Для табличных наборов применяется двухэтапный `train_test_split` с сохранением долей классов. Сначала отделяется 70% train, затем оставшиеся 30% делятся 1:1 на validation и test (15% каждый). Validation используется при необходимости ранней остановки в DL; для sklearn GridSearchCV основная оценка — внутренняя кросс-валидация на train. Test используется **один раз** для финальной отчётности, что предотвращает неявный p-hacking.

## Протокол подбора гиперпараметров

Сетки заданы в `config/config.yaml` и могут сужаться флагом `--quick` для CI. Для SVM применён RandomizedSearchCV с n_iter=15 из-за комбинаторного взрыва по C, gamma и kernel. Критерий отбора — weighted F1, согласованный с дисбалансом Covertype. Лучшие параметры логируются в results/metrics JSON.

## Протокол глубокого обучения

Fashion-MNIST: flatten 28×28, нормализация [0,1], val split 15% от train. CIFAR-10: нормализация пикселей, аугментация только на train graph (слои Keras внутри model). Оптимизатор Adam по умолчанию; scheduling через callback. Чекпоинты сохраняют лучшие веса по val_accuracy. Transfer: resize 224, нормализация ImageNet, два learning rate для head и full network.

## Оценка вычислительной сложности

Измеряется wall-clock время этапов transfer learning (head vs finetune) и косвенно — через сложность алгоритмов в теоретической таблице задачи 1. На CPU полный GridSearch на 15k Covertype занимает десятки минут; GPU ускоряет CNN/ResNet в 5–20 раз в зависимости от batch size.

## Документирование результатов

Каждый эксперимент порождает: (1) сериализованную модель, (2) JSON/CSV метрик, (3) PNG графики с понятными именами, (4) запись в лог-файл results/logs/. Это обеспечивает трассируемость от цифры в report.pdf до конкретного файла на диске — требование научной воспроизводимости.
